# 算法设计与分析A
第四学期必修课，3学分，计入保研绩点。据说计院这门课有上机作业。

首先我必须吐槽这门课的设计，究竟为何要在2024年选用一本2005年出版，且市面上早已很难买到的教材？有现成的、人人都学过的C/C++不用，为何要另外引入一门基于早已淘汰的Pascal语言简化设计的所谓“简洁明了”的SPARKS？这些徒然增加学生学习负担的东西，早就应该被扔进历史的垃圾堆了。

回到正题，这门课虽然看起来像是一门以写代码为主的课程，但它实际上只需要你背PPT就能拿到高分。相比面向对象、软件工程等背书的课程，这门课想拿到高分其实并不难，而且PPT也简洁明了，基本上全是干货，篇幅也不多，不必过于恐慌。

## 考核

- 部分老师会有隋唐小测，猫娘已将答案附在网盘资料中，供参考。

- 所有考试题目均不超出PPT范围（算法题除外，但也是那些算法）。也就是说，除了算法题，试卷上所有的题都是PPT出现过的理论或者举例。
- 证明题有范围。

## 知识点总结
1. 绪论：似乎没啥好讲的。
2. 导引、基本数据结构
   - 算法的五个特点，选择题可能会考
   - 时空复杂度的渐进表示，$O(n)$ $\Omega(n)$ $\Theta(n)$
   - 虽然PPT上花费了很大篇幅讲这几个表示符号的定义、性质、证明，但是似乎在考试中不怎么出现。
   - 其实该课程并不强制使用SPARKS答题，但是鉴于软件学院是一个以注释为基础的学院，用什么语言其实都差不多。
3. 分治
   - 分治算法的基本思想：分-治-合，适合采用递归实现。
   - 各个算法的抽象表示建议掌握，因为你在考试的时候写算法题用得上。
   - 二分查找问题：基本运算、二元比较树、以比较为基础查找的时间下界、时间复杂度分析
   - 归并排序：算法思想、时间复杂度分析、以比较为基础排序的时间下界
   - **斯特拉森矩阵乘法**：暴力方法、时间复杂度（具体实现没考过）
   - **二维极大点问题**：问题定义（什么是支配、什么是极大点）、暴力方法；设计思想（中位线）、优化方法（预排序）、时间复杂度分析
     - 怎么理解？其实就是：把一个大的点集合无限次切成两半（【12345678】->【1234】【5678】->【12】【34】【56】【78】->【1】【2】【3】【4】【5】【6】【7】【8】），最后切成一个一个的点，也就是每个集合只有一个点（按照定义，这个点就是这个集合的极大点了）。至于具体怎么分，先不要管。
     - 以上面的点1，2为例，假如点2支配点1，那么，在【12】这个上级的点集合里面，它的极大点就是点2。依次对其他6个点进行划分，假如我们在第3层得到的极大点是【2】【3】【6】【78】，那么【1234】的极大点就要在23里面产生，【5678】的极大点就会在678里面产生。对这个养蛊比赛重复进行，以此类推，这就是“治”和“合”的过程。为什么可以得到【78】？因为“支配”要求一个点必须在x和y上都大于另一个点的坐标，例如(2,1)和(1,2)就谁也支配不了谁。
     - 回到上面的问题，怎么分？这里采用的就是中位线。我们想要尽可能高效的把集合分开，减少不必要的比较，比如可以把x更小的提前淘汰掉，那么我们就可以在逻辑上画一条垂直于x轴的直线，把两边划成两个集合。也就是：确定一个x，把大集合分为两个。很自然的，我们会想到用一个中间点赋值给x，也就是最大值和最小值的平均。但是如果每次都要找两个最值，很麻烦，因此我们可以先对点集根据x的大小排序，这样直接取两端的平均值即可。
   - **主定理**：一定要会计算，不然你在考场上很难手推的
