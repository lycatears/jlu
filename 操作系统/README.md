## 知识点整理
- 各种系统举例都不用看。
- 操作系统概述
  - 相关基础概念：地位（与计算机硬件联系最紧密的一个系统软件，是对硬件的第一次扩充）、作用（管理资源、提供友好界面）、定义
  - 历史：手动操作，批处理系统（联机、脱机），执行系统（通道&中断，操作系统的初级阶段），多道批处理系统、分时系统（交互式任务）、实时处理系统
  - 特性：并发（注意与并行的区分）、共享（操作系统与用户程序共用资源）、异步（宏观上有多个程序同时运行，微观上是交替运行）、虚拟（一个物理实体变为多个逻辑实体）。需要注意这个“虚拟”不是我们日常使用的VMware等虚拟机，而是指由于多个程序共享硬件资源，在逻辑上似乎每个程序都有一自己的一套资源。
  - 分类
    - **多道批处理操作系统**：标志是脱机操作，适合处理运行时间较长的程序。处理的是**作业**（程序、数据、作业说明书），程序运行时无法实现交互式控制，含有输入井和输出井用于作业的调度。特性是多道（内存中同时存在多个作业）、成批（作业成批进入系统，由操作系统控制切换，无需用户干预）。
    - **分时操作系统**：标志是联机操作，适合程序的动态调试和修改。各用户通过终端与计算机连接（交互式），以时间片为单位交替、轮流为多个用户提供服务。特性：多路性（同时连接多个终端）、交互性（以对话的形式为用户服务）、独占性（每个用户都拥有一台虚拟机）。
    - **实时操作系统**：要求系统能对外部请求及时做出响应。特性：及时性（及时响应处理；同时处理多种实时任务时，应当保证所有任务都能在截止期限内处理完毕）、稳定性
    - 通用操作系统：同时包含批处理、分时、实时功能，三种任务的优先级一般是实时\>分时\>批处理。
    - 单用户、分布式、网络、多处理器、集群……后面这些了解一下就行了。
  - 硬件环境
    - 时钟 时钟中断
    - 堆（用户空间，保存动态变量）用户栈（用户空间，保存用户函数调用相关信息；传递系统调用参数）系统栈（又叫核心栈，保存系统调用之间的参数、返回值等信息；保存中断现场）
    - 寄存器：和计组差不多，就是多出来一个PSW
    - **处理器状态与特权指令**：处理器分为用户态（教材中称为目态）和系统态（核心态、管态），由PSW的程序状态字切换。只有系统态程序能执行特权指令，特权指令的执行会影响其他程序。用户态转换到系统态需要通过中断，例如时钟中断、系统调用；系统态转换到用户态需要修改PSW
    - 地址映射、存储保护措施（后续存储管理相关章节详解）
    - 中断装置：发现中断、响应中断
    - 管道与DMA
- 进程调度
  - 多道程序设计：操作系统最基本、最重要的技术。吞吐量=作业道数/处理时间。
    - 单道程序缺点：**设备资源利用率低**，各种设备资源往往不会总是处于忙碌状态，造成浪费；**内存资源利用率低**，单个程序的大小通常远小于内存容量；**处理器资源利用率低**，程序执行输入输出时处理器未被利用；**无法表达程序内部的并行性**。
    - 多道程序优点：上面那几个不好的地方变好了。
    - 多道程序带来的问题：上面三种资源的管理和分配问题。
  - 进程
    - **三种状态及其转换**：运行、就绪、等待；状态转换
    - 进程控制块：进程由进程控制块和程序两部分组成。进程控制块是进程存在的标志，存放在系统空间中。程序包含代码和数据。
    - 进程队列：分为就绪队列、等待队列、运行队列。就绪队列出队入队要按照调度算法；等待队列是每个等待事件有一个，进程等待事件发生时进入对应的等待队列，事件发生后进入就绪队列；运行队列是一个处理器一个，每个队列只有一个进程，指向运行队列头部的指针叫做运行指示字。


## 相关定义整理
> 仅收集课本中出现的相关定义，不是名词解释考试范围。
1. 操作系统：操作系统是位于硬件层之上、所有其他系统软件层之下的一个系统软件，通过它管理系统中的各种软件和硬件资源，使它们能被充分利用，方便用户使用计算机。
2. 进程：具有一定功能的程序关于一个数据集合的一次运行活动。
3. 进程控制块：标志进程存在的数据结构，其中包含系统对进程进行管理所需的全部信息。
4. 进程映像：进程的程序（代码和数据）被称为进程映像。
5. 系统开销：一般是指运行操作系统、对系统进行管理所花费的时间和空间。
