# 编译原理
开设于第6学期，通常在考试周前进行期末考试，课设答辩在考试前1-2周。
## 考试难度（22级）
较为简单，基本上没有难题。可以参考奔腾的题，做1-2套往年题基本就能掌握题型了。
## 课设
- 课设相对严格，需要讲清小组成员的分工合作，需要进行代码演示。可参考[snl-compiler](https://github.com/lycatears/snl-compiler)
- 祖传项目可供参考，但是最好有点自己的创新点，比如可视化分析之类的。
- 本人最终得分9/10，其他两个队友分别是8，7分。答辩时现场打分。
- 实验报告比较水，基本没人看。
## 建议
编译原理是最后一门考试课，考试难度不大，但是不好理解，学起来不太容易。由于编译原理不是考研408科目，市面上针对该课程的考试向视频资料极少。根据我自己的考试复习经验，考点总结如下：

- 绪论：概念性问题，一般在选择题考察。包括解释型语言、编译型语言，编译器的前端后端，高级语言生成目标程序的过程，编译程序的实现，还需要了解编译程序的构成部分。
- 形式语言与自动机：主要考点是正则表达式和有限自动机。文法考察概念性问题，一般是选择题或者简答题。正则表达式和有限自动机会在各种题型出没，必考大题，一般是先把正则表达式化为有限自动机，再把NFA化为DFA。
- 词法分析：考试中单独考察较少，但是重要的基础内容，一般在大题跟其他章节一起出现。课设可能会问相关知识。
- 语法分析：重难点，主要考察递归下降法和LL(1)分析法，自底向上不考。First，Follow，Predict三个集合的计算比较复杂，需要重点掌握。本章必考大题。
- 语义分析：必考大题。本章主要讲符号表，其实就是各种高级语言的变量作用域，刷两套真题就会做了，相对比较简单。
- 中间代码生成&中间代码优化：两章一般在一个大题里面考察，就是给你一段C语言或者Pascal代码，让你肉眼编译出四元组代码。语法制导可能会考小题。
- 运行时存储空间的组织与管理：其实还是作用域问题，考题一般是问同名的几个变量访问到哪一个，大题小题都有可能，但是比较简单。
- 目标代码生成：主要考小题，一般是概念性问题。
