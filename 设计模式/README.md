# 设计模式
必修课，2学分，计入保研绩点，开卷考试。可以认为是面向对象的后继课程。

## 复习指南
设计模式课程的内容框架非常清晰，主要难点在于为实际应用场景选择合适的设计模式。部分设计模式不做考察要求，下面的考点中不包含这些不考的内容。
- 以PPT为主，多过几遍PPT就行。涉及到概念性的大段文字，到考场上直接抄PPT就行了。
- 基础知识：设计模式的发展历史，面向对象设计原则（里氏替换原则，依赖倒置原则，开闭原则……），类之间的关系（依赖，组合，聚合，继承，实现……），UML类图
- 创建型模式：简单工厂模式，工厂方法模式，抽象工厂模式，建造者模式
- 结构型模式：原型模式，单例模式，适配器模式，桥接模式，组合模式，装饰器模式，外观模式，代理模式
- 行为型模式：职责链模式，命令模式，迭代器模式，中介者模式，备忘录模式，观察者模式，策略模式，状态模式，模板方法模式
- 另一种分类方法：类模式和对象模式
- 考试题量很大，而且需要写很多东西，题型参照14年样题，时间很紧张。
