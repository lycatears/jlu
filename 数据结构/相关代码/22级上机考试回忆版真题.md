## A 最大元素 20分
> Time Limit 50ms Memory Limit 30MB

- 给定n个正整数，编程求出其中的最大元素
- 输入格式：第一行是一个正整数n，第二行是n个正整数
- 输出格式：输出一个正整数，为最大元素
- 数据规模：n<=1000
- 测试点分值：2，3，5，5，5

## B 括号匹配 40分
> Time Limit 50ms Memory Limit 30MB
- 给定若干含有`{}[]()<>`四种括号的字符串，必须按照`{}→[]→()→<>`的次序嵌套（例如`{[<()>]}`是不匹配序列），不必判断表达式是否合法，不允许相同的括号嵌套（例如`(())`是不匹配序列）
- 输入格式：第一行是一个正整数n，随后n行是n个待处理的字符串
- 输出格式：每个字符串对应一行，若匹配成功输出Match，否则输出Fail
- 数据规模：n<=100，每个字符串长度不超过500
- 测试点分值：8，8，8，8，8

## C 最右侧表达式 40分
> Time Limit 50ms Memory Limit 30MB
- 给定一个字符串表示一个后缀表达式，其中小写字母代表操作数，大写字母代表运算符。表达式树的构建方法：从左到右依次扫描后缀表达式，操作数压栈，运算符连续弹栈两次分别作为该运算符节点的右、左孩子。输出表达式树的每层最右侧的节点。
- 输入格式：第一行是一个正整数n，随后n行是n个待处理的后缀表达式
- 输出格式：每个字符串输出一行，对应其最右侧表达式
- 数据规模：n<=100，每个字符串长度不超过500
- 测试点分值：8，8，8，8，8

## D 最长辗转子序列 50分
> Time Limit 50ms Memory Limit 30MB
- 给定一个字符串表示一个后缀表达式，其中小写字母代表操作数，大写字母代表运算符。定义“辗转子序列”：若序列$v_i,v_{i+1},...,v_j$满足：如果$v_k$是$v_{k-1}$的左孩子，则$v_{k+1}$是$v_k$的右孩子，如果$v_k$是$v_{k-1}$的右孩子，则$v_{k+1}$是$v_k$的左孩子，则是一个辗转子序列。$v_i$不一定是根节点，$v_j$不一定是叶子节点，编程求出表达式树中最长的辗转子序列的长度。
- 输入格式：第一行是一个正整数n，随后n行是n个待处理的后缀表达式
- 输出格式：每个字符串输出一个整数，对应其最长的辗转子序列的长度
- 数据规模：n<=100，每个字符串长度不超过500
- 测试点分值：10，10，10，10，10

## E 深度优先搜索序列 50分
> Time Limit 50ms Memory Limit 30MB
- 给定一个无向图和若干序列，判断这些序列是否是该有向图的深度优先搜索序列。
- 输入格式：第一行两个整数n，e，代表n个点e条边，后e行每行两个整数a，b，代表无向图的边，接下来一行是一个正整数m，最后m行每行是n个正整数，代表一个待判断的序列
- 输出格式：对于每个序列，如果是无向图的深度优先搜索序列，输出yes，否则输出no
- 数据规模：测试点0-3：n<10,e<50 测试点4：9000\<n , e \< 10000
- 测试点分值：10，10，10，10，10
